# Security Policy

## Reporting a Vulnerability

Do not open public issues for security-sensitive reports.

If this repository is hosted on GitHub, use GitHub's private vulnerability reporting or a private security advisory. If private reporting is not enabled, contact the repository maintainer through a private channel before disclosing details publicly.

## In Scope

Please report:

- committed credentials or secret material
- unsafe authentication or authorization defaults
- manifest issues that could expose data or grant unintended access
- accidental disclosure of internal infrastructure details

## Out of Scope

- general support requests
- environment-specific deployment failures caused by unreplaced placeholders
